Why we built it

Business problems solved

  • Siloed data across sales, procurement, warehouse, and finance departments
  • Disconnected communication between customer relations and product deal tracking
  • Manual calculations for complex import costing (landed cost, customs, margin)
  • Weak pipeline visibility lacking historical tracking
  • Open deals not programmatically compared against available warehouse stock
  • No cross-module reporting for executive leadership
  • Duplicated product data across multiple incompatible tools
  • Decisions being made without consolidated operational data

Architecture

How the platform is structured

FRONTEND — React 18 + TypeScript + Vite

Single-page application providing a unified workspace for CRM, PMS, Sales, Stock, Trade & Transit, and Reports. Styled with Tailwind CSS; communicates via REST and Supabase client.

BACKEND — FastAPI (Python) on Vercel

High-performance REST API handling CRM, PMS, sales pipeline, stock, integrated reports, and AI services — cross-module business logic and data orchestration.

DATA LAYER — Supabase (PostgreSQL + Auth + Storage)

Secure relational cloud database with row-level security, employee authentication, and file storage — structural integrity across customers, sales, inventory, and finance.

INTEGRATIONS

Google Gemini for AI (profiles, TDS extraction, summaries) · Supabase JWT and role-based access control (RBAC) for team access

Module deep-dive

Trade & Transit — Import procurement

Automated 4-stage costing flow

StageWhat is calculated
Capital Border capital outlay assessments (including parallel exchange metrics)
Customs Automated computation of CIF, duty, scanning, social taxes, WHT, and VAT
Landed Inland transportation factoring yielding net landed cost (ETB/kg)
Margin Target price generation, profit per kg, gross margin percentage, and revenue projections

Output integrates directly into executive snapshots, syncs with PMS pricing, and generates transit summaries.

Data model

  • Customer records map bi-directionally to sales pipelines and interaction logs
  • Product master data dictates categorical routing in the sales pipeline and import finance ledgers
  • Sales pipelines act as the central nervous system, connecting pricing structures, interaction histories, and stock ledgers
  • Stock inventories maintain strict relational integrity with the product master data and financial import records

End-to-end

Typical business workflows

A — Customer to closed deal System adds customer → auto-generates pipeline → logs interactions → advances stages → generates quotation → closes Won/Lost
B — Import procurement Request initiated → multi-stage calculation executed → data snapshotted → transit summary generated → pushed to executive reporting
C — Fulfillment check System analyzes open deal → compares required quantity against primary stock → triggers integrated risk report → logs paired stock movement upon fulfillment
D — Leadership reporting Customer metrics + stock availability + pipeline forecasting aggregated into PDF-ready executive BI dashboards

Roadmap
